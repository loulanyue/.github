# Support

Use GitHub Issues for reproducible project defects and scoped feature proposals. Before
opening an Issue, search existing reports and check the repository's README, roadmap,
release notes, and troubleshooting documentation.

These repositories are community-maintained and do not provide a commercial SLA.
Questions without a reproducible example may be converted to Discussions, redirected
to upstream documentation, or closed when they fall outside the project's scope.

Report vulnerabilities privately according to [SECURITY.md](SECURITY.md).
