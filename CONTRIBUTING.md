# Contributing

Thank you for helping improve a project maintained by `@loulanyue`.

## Before you start

1. Read the target repository's README, roadmap, license, and local contribution guide.
2. Search existing Issues and Pull Requests before opening a new one.
3. Discuss large features, public API changes, or broad refactors in an Issue first.
4. Never include secrets, private data, credentials, or machine-specific paths.

## Pull Requests

- Keep each Pull Request focused on one independently reviewable outcome.
- Explain the problem, approach, compatibility impact, and intentionally excluded scope.
- Include tests or reproducible validation evidence for behavior changes.
- Update documentation and changelog entries when users will notice the change.
- Identify copied or adapted work and preserve its authorship and license.
- Disclose material AI assistance and personally verify the submitted result.

Maintainers may ask for a smaller scope or additional evidence so the change can be
reviewed and maintained safely. Please follow the repository's code of conduct in all
project spaces.
